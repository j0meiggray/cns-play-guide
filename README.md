# cns-play-guide

## 项目简介

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面主要面向特定场景下的操作指南、信息展示或工具页面，不针对任何单个域名或具体网站。仓库内容以静态 HTML 文件为主，方便直接浏览或部署。

## 目录说明

```
cns-play-guide/
├── pages/            # 存放所有 HTML 页面文件
├── assets/           # 页面引用的资源文件（如图片、样式、脚本）
├── README.md         # 本文件
└── ...
```

- `pages/`：每个 HTML 文件独立存放，文件名与页面用途对应。
- `assets/`：按需组织子目录，避免资源文件分散。

## 页面归档说明

- 所有页面均为独立 HTML 文件，不依赖外部数据库或服务端环境。
- 页面内容可能包含操作指引、信息汇总或交互工具，具体用途见文件内注释。
- 页面之间无强关联，可单独使用或部署。
- 归档仅用于资料整理与分享，不保证页面内容的实时性或准确性。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来补充、修正页面内容。
- 提交新页面时，请将 HTML 文件放入 `pages/` 目录，并在文件开头添加简要注释说明用途。
- 如需添加资源文件，请放入 `assets/` 对应子目录，并在页面内使用相对路径引用。
- 本仓库不接受任何形式的商业推广、引流或违反平台规定的链接。

## 使用方式

- 直接打开 `pages/` 下的 HTML 文件即可浏览（支持本地浏览器或静态服务器）。
- 也可将整个仓库部署到任何静态托管服务（如 GitHub Pages、Netlify 等）。
